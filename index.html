    概述

    iziData提供如身份证实名认证、人脸比对识别、手持身份证识别、身份证OCR、个人/企业征信等API接口。

    iziData服务后台需要通过使用Access Key / Secret Key加密的方法来验证某个请求的发送者身份。Access Key（AK）用于标示用户，Secret Key（SK）是用户用于加密认证字符串和iziData服务用来验证认证字符串的密钥，其中SK必须保密，只有用户和iziData知道。

    SDK初始化，需要用户先申请到属于自己的AK和SK。

    请确保将YOUR_ACCESS_KEY和YOUR_SECRET_KEY替换为你自己的AK和SK。

    请确保将YOUR_ACCESS_KEY和YOUR_SECRET_KEY替换为你自己的AK和SK。

    请求者身份验证不通过将直接返回

    身份验证不通过应答样例

    {
  "status": "[错误状态码，详见后面表格]",
  "message": "出错信息"
}

    返回数据格式

    参数 描述
    status 状态码
    message 错误原因

    状态码说明

    status 说明
    INSUFFICIENT_BALANCE 余额不足
    INVALID_PARAMETER_ERROR 请求参数不合法
    EXPIRED_REQUEST 过期请求，请重试
    NO_AUTHORIZATION 验证不通过，用户不存在或者校验信息有误
    RETRY_LATER 系统错误

    HTTP请求

    POST https://api.izi.credit/v1/identitycheck

    请求参数

    参数 描述
    name 可选，字符串，认证人员的姓名
    id 字符串，认证人员的身份证号NIK，格式为16位阿拉伯数字

    返回数据格式

    参数 描述
    status 状态码
    message id：身份证号码，NIK
    name：姓名
    province：省份，可能为空
    city：城市，可能为空
    district：区县，可能为空
    village：村镇，可能为空

    状态码说明

    status 说明
    OK 计费，查找成功，库中存在查询的NIK，且姓名比对一致
    NOT 计费，查找成功，库中存在查询的NIK，但姓名比对不一致
    PERSON_NOT_FOUND 免费，我们库中查不到此NIK
    INVALID_ID_NUMBER 免费，输入的NIK不合法
    RETRY_LATER 免费，系统错误，请稍后重试

    HTTP请求

    POST https://api.izi.credit/v1/mutiphone

    请求参数

    参数 描述
    id 必选，字符串，查询人员的身份证号
    phone 可选，字符串，查询人员的电话号

    返回数据格式

    参数 描述
    status 状态码
    message id_info：查询身份证号码下面的手机号
    phone_info：查询手机号下的相关信息
    id：查询手机号所属的身份证号
    phone：查询手机号所属身份证号下的手机号信息

    状态码说明

    status 说明
    OK 查询成功计费，message为查询结论
    INVALID_PHONE_NUMBER 免费，输入的电话号码不合法
    INVALID_ID_NUMBER 免费，输入的身份证号码不合法
    NOT_FOUND 免费，未查询到结果
    RETRY_LATER 免费，系统错误，请稍后重试
